# Тестовая итерация мобильного контроллера

Документ фиксирует, что было сделано в тестовой ветке работ по локальному сайту для телефонов. Изменения считаются экспериментальными: если поведение не подойдёт, их можно откатить одним коммитом.

## Что изменено

- Старый монолитный `Assets/StreamingAssets/WebClient/app.js` удалён.
- Старый монолитный `Assets/StreamingAssets/WebClient/style.css` удалён.
- `index.html` переписан как каркас локального мобильного сайта.
- Web-клиент разнесён на папки:
  - `css/`
  - `js/`
  - `js/screens/`
  - `js/components/`
- Добавлен экран регистрации игрока по имени и фото.
- Добавлено восстановление сессии телефона через `localStorage`.
- Добавлено мобильное лобби со списком игроков.
- Добавлен экран импорта существующего персонажа.
- Добавлена заготовка экрана мобильного редактора персонажа.
- Добавлен игровой экран с HP, бронёй, движением, бросками, активным оружием и кнопками управления.

## Серверные изменения

В `WebServerManager` добавлены тестовые API:

- `POST /api/session/register` - зарегистрировать игрока без создания персонажа.
- `GET /api/session/restore?playerId=...` - восстановить телефон после перезагрузки страницы.
- `GET /api/lobby/state` - получить состояние лобби и игроков.
- `GET /api/characters` - получить список сохранённых персонажей.
- `POST /api/character/import` - привязать существующего персонажа к игроку.
- `POST /api/player/ready` - установить готовность игрока.
- `POST /api/camera/focus` - переключить player view камеру на игрока.

Также расширен старый `GET /api/game/state?playerId=...`. Теперь он отдаёт:

- HP;
- броню;
- очки движения;
- броски;
- активное оружие;
- флаг активного хода игрока;
- флаг активного боя;
- статусы;
- ближайшие цели;
- prompt перехода между картами.

## Движение с телефона

`POST /api/action/move` переделан:

- ищет только реальную runtime-фишку игрока, а не player view clone;
- в бою разрешает движение только активной фишке;
- списывает очки движения в бою;
- обновляет `gridPosition`;
- обновляет `transform.position`;
- синхронизирует позицию игрока в `CampaignGameSession`;
- обновляет GM UI после движения.

## Сессия игрока

В `CampaignGameSession` добавлено состояние игрока без персонажа:

- `AddRegisteredPlayer`;
- `AssignCharacterToPlayer`;
- `SetPlayerReady`;
- поле `isReady`.

Это нужно для схемы: сначала игрок подключается по телефону, потом выбирает или создаёт персонажа.

## Player View

`MainMenuPlayerViewManager` теперь может показывать фото зарегистрированного игрока даже до выбора персонажа.

`CampaignGameLoader` получил публичный метод `FocusPlayerViewOnPlayer`, чтобы телефон мог переключать камеру на своего игрока.

## Что пока не сделано

- Полноценный мобильный редактор персонажа.
- Настройка оружия и способностей с телефона.
- Инвентарь.
- Dice-widget для атаки и защиты.
- Двухфазная атака: запрос атаки, бросок игрока, отправка результата, расчёт Unity.
- Перебросы за монеты.
- Полная проверка UI на реальных телефонах.
