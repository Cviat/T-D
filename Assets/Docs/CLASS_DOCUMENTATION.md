# Class Documentation

Этот файл фиксирует назначение классов проекта. При добавлении нового класса нужно кратко описывать:

- что делает класс;
- зачем он нужен;
- где используется;
- какие важные решения или ограничения у него есть.

## UI

### MainMenuAction

**Файл:** `Assets/RPGTable/UI/MainMenuAction.cs`

Enum со списком действий главного меню: старт игры, продолжение, добавление контента и выход.

Нужен, чтобы кнопки меню не хранили строковые команды. Кнопка выбирает значение enum, а `MainMenuController` решает, что именно делать.

### MainMenuController

**Файл:** `Assets/RPGTable/UI/MainMenuController.cs`

Главный обработчик действий меню и переключения панелей.

Сейчас умеет:

- запускать сцену стола `RPGTablePrototype`;
- обрабатывать кнопку продолжения тем же переходом, что и старт;
- скрывать главное меню и показывать панель добавления контента;
- открывать сцену редактора карты по кнопке `Создать карту`;
- принимать заглушку `Создать фишку`;
- возвращаться из панели добавления контента назад в главное меню;
- выходить из приложения или из Play Mode в редакторе.

Нужен как единая точка поведения главного меню. Когда появятся сохранения, выбор кампаний и импорт контента, логика будет расширяться здесь или выноситься в отдельные сервисы.

### MainMenuButton

**Файл:** `Assets/RPGTable/UI/MainMenuButton.cs`

Компонент для кнопки главного меню.

Хранит действие `MainMenuAction`, находит `MainMenuController`, вызывает нужное действие по клику и меняет цвет кнопки при наведении.

Нужен, чтобы кнопки в сцене оставались простыми UI-объектами, но не требовали ручной настройки `OnClick` в инспекторе для каждой сцены.

### MainMenuFitToSafeArea

**Файл:** `Assets/RPGTable/UI/MainMenuFitToSafeArea.cs`

Компонент для адаптации UI-панели под `Screen.safeArea`.

Сейчас не используется в сцене главного меню, потому что первая версия ориентирована на desktop/TV. Оставлен как готовый небольшой компонент для будущего телефонного UI или экранов, где важны вырезы и системные зоны.

### PrototypeHud

**Файл:** `Assets/RPGTable/UI/PrototypeHud.cs`

Временный debug-HUD для сцены стола.

Показывает текущий режим `Master/Player`, короткие подсказки и список инициативы. Это не финальный UI, а быстрый диагностический слой для раннего прототипа.

После появления нормального UI стола этот класс нужно удалить или заменить полноценными панелями мастера.

## Editor

### RPGTableMapEditorBuilder

**Файл:** `Assets/RPGTable/Editor/RPGTableMapEditorBuilder.cs`

Editor-команда `RPG Table > Build Map Editor Scene`.

Создает первую сцену редактора карт: большое серое поле 100x100 клеток, видимую сетку, камеру с управлением мышью, левую панель элементов, нижнюю панель и кнопку импорта изображений.

## MapEditor

### UserElementAssetStore

Хранит импортированные пользователем изображения в `Application.persistentDataPath/RPGTable/UserElements`, чтобы они оставались после перезапуска.

### MapEditorElementPalette

Заполняет левую панель импортированными изображениями и позволяет выбрать элемент для размещения.

### MapEditorElementSpawner

Создает выбранный элемент на карте по клику.

### PlacedMapElement

Позволяет двигать размещенный элемент мышью и менять его размер колесом мыши.

## Board

### BoardGridVisual

**Файл:** `Assets/RPGTable/Board/BoardGridVisual.cs`

Рисует видимое игровое поле: серый фон и линии клеток. Используется и в редакторе карт, и в обычной игровой сцене.

## Input

### MouseCameraController

**Файл:** `Assets/RPGTable/Input/MouseCameraController.cs`

Двигает ортографическую камеру мышью: удержание правой или средней кнопки перетаскивает карту, колесо меняет масштаб.

Компонент общий для редактора карт и будущего игрового стола.

### RPGTableMainMenuBuilder

**Файл:** `Assets/RPGTable/Editor/RPGTableMainMenuBuilder.cs`

Editor-команда `RPG Table > Build Main Menu Scene`.

Автоматически собирает сцену главного меню:

- импортирует фон как Sprite;
- импортирует отдельные PNG кнопок как Sprite с прозрачностью;
- импортирует фон панели добавления контента;
- создает `Canvas`;
- ставит фон из приложенного изображения;
- добавляет затемнение слева для читаемости;
- создает декоративный логотип;
- создает четыре настоящие UI-кнопки из готовых изображений: старт, продолжение, добавление контента, выход;
- создает панель добавления контента с кнопками `Создать карту`, `Создать фишку`, `Назад`;
- добавляет `EventSystem`;
- добавляет сцены меню и стола в Build Settings.

Нужен, чтобы меню можно было пересобрать одним действием и не зависеть от ручной настройки UI в инспекторе.

### RPGTablePrototypeBuilder

**Файл:** `Assets/RPGTable/Editor/RPGTablePrototypeBuilder.cs`

Editor-команда `RPG Table > Build Prototype Scene`.

Собирает первую тестовую сцену стола: поле, токены, туман войны, камеру, свет, HUD и тестовые карты способностей.

Нужен как быстрый способ пересоздать вертикальный срез во время разработки.
